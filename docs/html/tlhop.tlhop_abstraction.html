.tlhop_extension.parser_complex_column(input_col, output_col=None)[source]#

Because of their information complexity, some columns saved in string type are actually a JSON information. Saving in a format already structured in Spark (Struct) would make the final dataset schema very large, and would have a significantly higher storage cost. As not all columns are used in a single query, we persist such columns in string and create an API containing the schema for easy structuring, when necessary. This method can be used to parser these type of columns during execution.

Parameters:

  • input_col – Column name to be parsed;

  • output_col – Output column name. By default, it will add a suffix “_parsed” in the original name;

Returns:

Spark DataFrame
